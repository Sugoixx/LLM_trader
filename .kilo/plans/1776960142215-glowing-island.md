# Plan: Optimisation Rentabilité CRUDOIL & CRYPTO

## Objectif
Maximiser la rentabilité du bot de trading avec un focus sur CRUDOIL et CRYPTO. Chaque trade doit être gagnant par une sélection rigoureuse des signaux et une gestion des risques optimisée.

## Analyse des Problèmes Identifiés

### 1. Stratégies Algo - Faiblesses Critiques
| Stratégie | Problèmes | Impact |
|-----------|-----------|--------|
| MA Crossover | BUG min_bars (ligne 32), pas de filtre ADX, pas de volume, confiance fixe 0.60 | **HIGH** - Signaux précoces, trades en range |
| RSI Crossover | RSI(14) trop lent pour crypto, pas de filtre ADX, confiance fixe 0.65 | **MEDIUM** - Whipsaws en trends forts |
| Bollinger Reversion | Manque de breakout trades | MEDIUM |

### 2. Gestion des Risques - Manquements
- Position sizing fixe (3%/2%/1%) ignorant la volatilité
- Pas de trailing stop dynamique basé sur la performance
- SL/TP par défaut (2x/4x ATR) inadaptés pour CRUDOIL/CRYPTO

### 3. Spécificités CRUDOIL & CRYPTO
- CRUDOIL: Seuil volatilité 2.5x (COMMODITIES) trop bas, devrait être 4.0x+
- CRYPTO: Seuil 3.0x peut être insuffisant pour l'altcoin volatility
- Pas de gestion des heures de marché pour l'oil

```python
# Nouveau: Pause après drawdown de 5%
MAX_DRAWDOWN_PCT: float = -5.0

def _check_drawdown(self, current_capital, initial_capital) -> Optional[str]:
    drawdown_pct = ((current_capital - initial_capital) / initial_capital) * 100
    if drawdown_pct < self.MAX_DRAWDOWN_PCT:
        return f"Fast trader BLOCKED: Drawdown {drawdown_pct:.1f}% exceeds {self.MAX_DRAWDOWN_PCT}% limit"
    return None
```
